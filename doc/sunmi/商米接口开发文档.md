**旧版 vs 新版打印方案对比**：

| 对比项 | 旧版 ExtPrinterService AIDL | 新版 PrinterSdk lineApi |
|--------|---------------------------|------------------------|
| 绑定方式 | 手动 AIDL 绑定 `com.sunmi.extprinterservice` | `PrinterSdk.getInstance().getPrinter()` |

| 事务模式 | 无 | `enableTransMode` + `printTrans` |
| 推荐度 | 旧方案，兼容老设备 | **推荐**，新版接口更丰富 |

> **本项目选择新版 lineApi**。如遇兼容问题，可降级使用旧版 ExtPrinterService AIDL。

## 三、扫码功能（扫码头引擎 — 红外线扫码）

### 3.1 方案选择

**选扫码头引擎方式**（红外线扫码），理由：
- V3PLUS 带有硬件扫码头（红色激光线 + 中心瞄准点），点击扫码按钮后出现红色激光，像扫描枪一样连续扫
- 扫码头是免开发组件，通过广播输出扫码结果，无需打开摄像头预览界面
- 扫码体验：按键 → 红色激光亮 → 扫到码 → 提示音 → 自动继续扫 → 连续不停
- 不需要 `startActivityForResult`，不需要扫码界面，纯后台接收广播即可

> **与摄像头扫码的区别**：
> - 摄像头扫码（方式一）：需要打开一个扫码界面，用户看到摄像头预览，扫码后界面关闭。参考 `摄像头扫码SDK说明.md`
> - 扫码头引擎（本项目选择）：不需要界面，红色激光直接扫，结果通过系统广播自动返回到 App

### 3.2 AndroidManifest.xml 配置

> **当前状态**：尚未添加 `<queries>` 声明，需要添加。

### 3.3 扫码输出方式

扫码头引擎提供三种接收方式，本项目用**广播输出方式**（最简单、最精确）：

| 输出方式 | 说明 | 适用场景 |
|---------|------|---------|
| 模拟键盘 | 扫码结果模拟按键事件，自动输出到焦点框文本 | 不需要写代码，但无法区分扫码和手动输入 |
| 直接填充 | 扫码结果拷贝到剪贴板，自动输出到焦点框文本 | 同上，速度更快 |
| **广播输出** | 扫码结果作为系统广播发送，App 注册接收器获取 | **本项目选用**：可区分扫码/手动输入，精确控制业务逻辑 |

> **切换输出方式**：在商米设备"系统设置 → 扫码设置"中可以切换输出方式。选择"广播输出"。
> 也可以通过 AIDL 接口 `setScanMode()` 代码切换，但一般用系统设置即可。

### 3.4 AIDL 接口定义

**文件路径**：`app/src/main/aidl/com/sunmi/scanner/IScanInterface.aidl`

```aidl
package com.sunmi.scanner;

import android.view.KeyEvent;

interface IScanInterface {
    void sendKeyEvent(in KeyEvent key);   // 自定义按键触发扫码
    void scan();                           // 触发开始扫码
    void stop();                           // 触发停止扫码
    int getScannerModel();                 // 获取扫码头类型
}
```

> **权威来源**：商米官方 `扫码头开发及使用手册文档`（2019 版，v2.3.x）第 2.3 节。
> **注意**：此接口与商米扫码头引擎服务 **严格一一对应**，不可随意增删方法。AIDL 的方法交易码按声明顺序分配，多写/少写都会导致 Binder 通讯错位。
> **历史上踩过的坑**：之前项目中添加了 `initScanner`/`setScanMode`/`setScanSound` 等自创方法，导致 `scan()` 实际调用了服务端的 `stop()`，扫码按钮点了就闪退。详见 README 5.14 节。

### 3.5 服务绑定

扫码服务通过 AIDL 绑定。**服务 Action**：`com.sunmi.scanner.IScanInterface`，**服务包名**：`com.sunmi.scanner`。

```kotlin
private var scanInterface: IScanInterface? = null

private val scanConn = object : ServiceConnection {
    override fun onServiceConnected(name: ComponentName?, service: IBinder?) {
        scanInterface = IScanInterface.Stub.asInterface(service)
    }
    override fun onServiceDisconnected(name: ComponentName?) {
        scanInterface = null
    }
}

private fun bindScannerService() {
    val intent = Intent()
    intent.setPackage("com.sunmi.scanner")
    intent.setAction("com.sunmi.scanner.IScanInterface")
    bindService(intent, scanConn, Context.BIND_AUTO_CREATE)
}

private fun unbindScannerService() {
    try { unbindService(scanConn) } catch (_: Exception) {}
    scanInterface = null
}
```

> **权威来源**：商米官方 `扫码头开发及使用手册文档` 第 2.2 节。

> **绑定时机**：`onCreate()` 中绑定，`onDestroy()` 中解绑。
> **绑定失败处理**：如果设备不带扫码头，`scanInterface` 保持 null，调用 `scan()` 时 catch `RemoteException`。

### 3.6 广播接收扫码结果

**广播 Action**：`com.sunmi.scanner.ACTION_DATA_CODE_RECEIVED`

**广播参数**：

| Intent Extra | 类型 | 说明 |
|-------------|------|------|
| `data` | String | 扫码结果内容（UTF-8 字符串） |
| `label` | String | 条码类型（QR_CODE、CODE128、EAN-13 等） |

**广播接收器代码**：

```kotlin
inner class ScanReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        if ("com.sunmi.scanner.ACTION_DATA_CODE_RECEIVED" == intent.action) {
            val code = intent.getStringExtra("data") ?: ""
            val label = intent.getStringExtra("label") ?: ""  // 条码类型，本项目不使用
            if (code.isNotEmpty()) {
                addByCode(code)  // 喂给已有的业务逻辑方法
            }
        }
    }
}
```

**广播注册/注销**：

```kotlin
private var scanReceiver: ScanReceiver? = null

private fun registerScanReceiver() {
    scanReceiver = ScanReceiver()
    val filter = IntentFilter("com.sunmi.scanner.ACTION_DATA_CODE_RECEIVED")
    registerReceiver(scanReceiver, filter)
}

private fun unregisterScanReceiver() {
    try { unregisterReceiver(scanReceiver) } catch (_: Exception) {}
    scanReceiver = null
}
```

> **注册方式**：必须使用动态注册（`registerReceiver`），不使用静态注册（AndroidManifest）。
> **原因**：动态注册只在 Activity 存活时接收广播，退出后自动停止。避免 Activity 销毁后仍收到扫码广播。

### 3.7 软触发扫码（控制红色激光）

```kotlin
private var isScanning = false  // 当前是否在连续扫码状态

private fun startScan() {
    isScanning = true
    b.btnScan.text = "停止扫码"
    try {
        scanInterface?.scan()       // 软触发：启动扫码头，红色激光亮起
    } catch (e: RemoteException) {
        e.printStackTrace()
    }
}

private fun stopScan() {
    isScanning = false
    b.btnScan.text = "扫码"
    try {
        scanInterface?.stop()       // ⚠️ 官方接口叫 stop()，不是 stopScan()
    } catch (e: RemoteException) {
        e.printStackTrace()
    }
}

private fun onScanClick() {
    if (isScanning) stopScan() else startScan()
}
```

> **扫码灯自动联动**：`scan()` → 红色激光自动亮，`stop()` → 红色激光自动灭。无需单独控制灯。
> **方法名注意**：官方 AIDL 接口定义的方法名是 `stop()` 不是 `stopScan()`，代码中调用时要用 `scanInterface?.stop()`。之前写错导致通讯错位。

### 3.8 连续扫码说明

- 扫码头引擎天然支持连续扫码：激光一直亮着，扫到一个码 → 发广播 → 继续扫下一个
- 不需要像摄像头方式那样设置 `SCAN_MODE=true`
- 每扫到一个码就发一次广播 → `ScanReceiver.onReceive` → `addByCode(code)` → 自动验证+加入列表
- 用户点击"停止扫码"时调 `stop()` 关闭激光，回到新增出库页面

### 3.9 提示音和震动

商米扫码头自带扫码成功提示音和震动，**无需代码控制**。在设备"系统设置 → 扫码头设置 → 提示方式"中可开关。

AIDL 接口中没有 `setScanSound`/`setScanVibrate` 方法（之前版本自行添加的方法不存在于官方接口中）。

## 四、打印功能（热敏小票 lineApi）

> **官方来源**：商米打印 SDK 文档（docs.sunmi.com → 集成开发指南 → 打印 → 热敏小票接口）
> **本地副本**：`/workspace/doc/sunmi/打印热敏小票接口.md`

### 4.1 方案选择

**选 lineApi**（热敏小票行式打印接口），理由：
- V3PLUS 内置热敏打印机（支持 58mm/80mm 纸卷，最大 80mm），适合小票打印
- lineApi 以行为单位逐行排版输出，适合本项目"表头 + 明细列表"的排版需求
- `printTexts()` 支持多列排版，可直接用于明细列表的序号/条码/品名/规格分列
- canvasApi 是标签画布式接口，适合图文混排标签，本项目不需要

### 4.2 依赖配置

**新版 PrinterSdk 无需额外 gradle 依赖**。商米 V3PLUS 系统内置打印服务（`woyou.aidlservice.jiuiv5`），新版 PrinterSdk 通过 AIDL 自动绑定。

> **当前项目 build.gradle 路径**：`/workspace/app/build.gradle`
> **当前状态**：无商米打印 SDK 依赖。实现时先尝试直接使用 PrinterSdk，如失败再添加 SDK jar/aar。

如需添加依赖（备选）：

### 4.7 打印排版设计

**打印触发条件**：出库单/退库单 且 状态=已出库/已退库 时显示打印按钮。

**数据来源**（来自 GoodlistActivity）：

| 数据 | 变量 | 字段 |
|------|------|------|
| 表头 | `orderhead: JSONObject?` | CusCode, name, MakeDate, Status, BillType |
| 明细 | `list: MutableList<JSONObject>` | RowNumber, ScanCode, Name, Spec, Spec2 |

**排版方案**：

```
┌──────────────────────────────────┐
│          美丰客户中心              │  ← 居中加粗大字 (setTextSize=36, enableBold)
│         出库单/退库单              │  ← 居中加粗 (setTextSize=30, enableBold)
├──────────────────────────────────┤  ← DOTTED 分隔线
│ 客户代码: XXXXX                   │  ← 左对齐 (setTextSize=24)
│ 客户名称: XXXXX                   │
│ 日期: 2026-06-15                  │
│ 类型: 出库单                       │
│ 状态: 已出库                       │
├──────────────────────────────────┤  ← DOTTED 分隔线
│ 序号  条码        品名   规格      │  ← printTexts 分4列, 比例 1:2:2:2
│  1   SC001      产品A  500ml     │  ← 每行一条明细
│  2   SC002      产品B  250ml     │
│  3   SC003      产品C  1L        │
│ ...                               │
├──────────────────────────────────┤  ← DOTTED 分隔线
│ 合计: N 件                         │  ← 左对齐加粗
│                                   │  ← 空行（方便撕纸）
└──────────────────────────────────┘  ← autoOut 走纸+切纸
```

**58mm/80mm 纸宽排版说明**：
- V3PLUS 支持两种纸宽：58mm（可打印宽度约 **384 像素**）和 80mm（可打印宽度约 **576 像素**）
- 使用比例排版自动适配，`printTexts` 的 `colsWidthArr` 按比例分配宽度，如 `intArrayOf(1, 2, 2, 2)` 表示 1:2:2:2，58mm 下 = 384/7 ≈ 55:110:110:110 像素，80mm 下 = 576/7 ≈ 82:165:165:165 像素
- 中文字符约 24 像素宽（setTextSize=24 时），一列 110 像素约 4~5 个汉字

### 4.8 打印代码示例

```kotlin
private fun printReceipt() {
    val printer = PrinterSdk.getInstance().getPrinter()
    val lineApi = printer.lineApi()
    val head = orderhead ?: return

    // 开启事务模式（全部排版完再一次性打印）
    lineApi.enableTransMode(true)

    // ── 表头标题 ──
    lineApi.initLine(BaseStyle.getStyle().setAlign(Align.CENTER))
    lineApi.printText("美丰客户中心", TextStyle.getStyle().enableBold(true).setTextSize(36))
    lineApi.printText(
        head.optString("BillType", ""),
        TextStyle.getStyle().enableBold(true).setTextSize(30)
    )

    // ── 分隔线 ──
    lineApi.printDividingLine(DividingLine.EMPTY, 10)
    lineApi.printDividingLine(DividingLine.DOTTED, 2)
    lineApi.printDividingLine(DividingLine.EMPTY, 10)

    // ── 客户信息 ──
    lineApi.initLine(BaseStyle.getStyle().setAlign(Align.LEFT))
    lineApi.printText("客户代码:${head.optString("CusCode", "")}", TextStyle.getStyle())
    lineApi.printText("客户名称:${head.optString("name", "")}", TextStyle.getStyle())
    lineApi.printText("日期:${head.optString("MakeDate", "")}", TextStyle.getStyle())
    lineApi.printText("类型:${head.optString("BillType", "")}", TextStyle.getStyle())
    lineApi.printText("状态:${head.optString("Status", "")}", TextStyle.getStyle())

    // ── 分隔线 ──
    lineApi.printDividingLine(DividingLine.EMPTY, 10)
    lineApi.printDividingLine(DividingLine.DOTTED, 2)
    lineApi.printDividingLine(DividingLine.EMPTY, 10)

    // ── 明细表头 ──
    val headerStyle = TextStyle.getStyle().enableBold(true).setAlign(Align.CENTER)
    lineApi.printTexts(
        arrayOf("序号", "箱码", "产品名称", "规格", "规格2"),
        intArrayOf(1, 2, 2, 1, 1),
        arrayOf(headerStyle, headerStyle, headerStyle, headerStyle)
    )

    // ── 明细列表（每个序号一行） ──
    val rowStyle = TextStyle.getStyle().setAlign(Align.CENTER)
    for (i in list.indices) {
        val item = list[i]
        lineApi.printTexts(
            arrayOf(
                item.optString("RowNumber", "${i + 1}"),
                item.optString("ScanCode", ""),
                item.optString("Name", ""),
                item.optString("Spec", "") + " " + item.optString("Spec2", "")
            ),
            intArrayOf(1, 2, 2, 2),
            arrayOf(rowStyle, rowStyle, rowStyle, rowStyle)
        )
    }

    // ── 合计 ──
    lineApi.printDividingLine(DividingLine.EMPTY, 10)
    lineApi.printDividingLine(DividingLine.DOTTED, 2)
    lineApi.printDividingLine(DividingLine.EMPTY, 10)
    lineApi.printText("合计: ${list.size} 件", TextStyle.getStyle().enableBold(true))

    // ── 结尾空行（方便撕纸）──
    lineApi.printDividingLine(DividingLine.EMPTY, 30)

    // ── 走纸到出口 + 自动切纸 ──
    lineApi.autoOut()

    // ── 提交事务 ──
    lineApi.printTrans(object : PrintResult() {
        override fun onResult(resultCode: Int, message: String?) {
            lineApi.enableTransMode(false)  // 必须关闭事务模式！
            if (resultCode == 0) {
                MfUi.toast(this@GoodlistActivity, "打印成功")
            } else {
                MfUi.toast(this@GoodlistActivity, "打印失败: ${message ?: "未知错误"}")
            }
        }
    })
}
```

## 六、注意事项和常见问题

### 6.1 扫码相关

1. **AIDL 服务绑定时机**：必须在 `onCreate` 中绑定扫码服务，`onDestroy` 中解绑。如果绑定失败（设备不带扫码头），`scanInterface` 为 null，`scan()` 调用会 catch RemoteException。
2. **广播注册方式**：使用动态注册（`registerReceiver`），不使用静态注册（AndroidManifest）。动态注册只在 Activity 存活时接收广播，退出后自动停止。
3. **扫码灯自动联动**：扫码头引擎的红色激光跟随扫码状态自动亮灭。`scan()` → 激光亮，`stopScan()` → 激光灭。无需单独控制。
4. **提示音**：商米系统自带扫码成功提示音，`setScanSound(true)` 开启。没有失败提示音。
5. **连续扫码**：扫码头天然连续扫码，激光一直亮着，每扫到一个码发一次广播。不需要像摄像头方式设置 `SCAN_MODE`。
6. **输入法冲突**：已知百度小米版输入法会影响扫码头结果输出。建议使用谷歌拼音输入法或关闭输入法影响。
7. **输出模式**：如果广播收不到扫码结果，检查商米设备"系统设置 → 扫码设置"中输出模式是否为"广播输出"。
8. **全局物理按键拦截**：`BaseActivity.dispatchKeyEvent` 拦截全部物理按键，只放行以下按键：
   - `KEYCODE_BACK`（返回键）— 正常返回
   - `KEYCODE_VOLUME_UP`（音量+）— 调音量
   - `KEYCODE_VOLUME_DOWN`（音量-）— 调音量
   
   其余所有物理按键（包括 `DPAD_CENTER`/`DPAD_DOWN`/`DPAD_UP`/`ENTER`/`NUMPAD_ENTER` 等）一律消费掉不传到 UI 层。物理扫码走硬件通路（激光→广播→`ScanReceiver`），无需按键事件配合。详见 `BaseActivity.kt`。
9. **AIDL 接口必须与官方严格一致**：商米扫码头引擎 AIDL 只有 4 个方法（`sendKeyEvent`/`scan`/`stop`/`getScannerModel`），不要自行添加 `initScanner`/`setScanMode`/`setScanSound` 等方法。AIDL 方法交易码按声明顺序分配，多写/少写会导致 Binder 通讯错位，`scan()` 可能实际调用 `stop()`。历史教训见 README 5.14 节。

### 6.2 打印相关

8. **事务模式陷阱**：开启 `enableTransMode(true)` 后若忘记 `printTrans` 或异常退出，后续非事务打印会不打印。官方文档特别强调：
   - 在 `printTrans` 回调中必须调用 `enableTransMode(false)`
   - 在 `onDestroy` 中也调用 `enableTransMode(false)` 兜底
   - **非事务打印前也建议先调一次 `enableTransMode(false)`**，防止因之前异常退出事务未关闭导致不打印
9. **58mm/80mm 纸宽适配**：V3PLUS 支持 58mm 和 80mm 纸卷，排版时使用比例排版（`colsWidthArr` 按比例分配宽度）自动适配，不要写死像素值。58mm 可打印宽度约 384 像素，80mm 约 576 像素。
10. **分隔线兼容性**：老版本打印服务中 `SOLID` 和 `DOTTED` 可能不完美支持，需联系商米技术支持更新到最新版打印服务。
11. **图片打印背景**：图片需非透明背景，否则二值化为黑色覆盖原图。
12. **打印服务版本**：部分功能（如自定义条码大小、列内样式）需要打印服务 6.0.0+ 或 6.5.0+ 版本。可通过设备中"硬件管家"查看打印服务版本。
13. **自定义字体**：如需使用自定义字体，将 `.ttf` 字体文件放在 App 的 `assets` 目录中，通过 `TextStyle.setFont("字体名")` 设置。外接非商米打印机不支持。
14. **PrinterSdk 找不到**：如果 `PrinterSdk` 类无法引用，需要从商米开发者网站下载打印 SDK aar 并添加到项目依赖中。

### 6.3 降级方案

15. **打印降级**：如新版 PrinterSdk/lineApi 无法使用，可降级为旧版 `ExtPrinterService.aidl`（见 2.4 节）。绑定方式：
    ```kotlin
    val intent = Intent()
    intent.setPackage("woyou.aidlservice.jiuiv5")
    intent.setAction("woyou.aidlservice.jiuiv5.IPrinterService")
    bindService(intent, conn, Context.BIND_AUTO_CREATE)
    ```
16. **扫码降级**：~~如扫码头引擎不支持（设备无扫码头），可降级为摄像头扫码方式~~。**已确认：不做扫码降级**，V3PLUS 有扫码头，只实现扫码头引擎方案。摄像头扫码计数功能也暂不实现。

## 九、需求确认 Q&A（2026-06-15）

> 以下为开发前与用户确认的需求细节，实现时严格按此执行。

### 扫码相关

**Q1：扫码按钮行为？单次还是连续？**
> **A：连续模式。** 点击"扫码"按钮后，红色激光持续扫描，扫到一个码立即处理，然后继续扫描下一个，直到用户点击"停止扫码"按钮。

**Q2：扫码期间能否手动输入？**
> **A：不可以。** 扫码期间禁用手动输入框。具体交互：
> - 点击"扫码"按钮后 → 输入框获取焦点时提示"请停止扫码再使用录入功能"
> - "扫码"按钮变为"停止扫码"按钮，颜色变红
> - 确认/取消按钮、粘贴导入按钮等均禁用
> - 点击"停止扫码"后 → 所有按钮恢复初始状态，输入框可用

**Q3：扫码重复码如何提示？**
> **A：弹窗提示。** 扫到重复码时弹出提示，用户点击确认后**不需要再次点击扫码**，激光继续扫描，可以继续扫下一个码。只有用户主动点击"停止扫码"才会停止。

**Q3.5：是否做扫码降级？**
> **A：不做。** V3PLUS 有扫码头，只实现扫码头引擎方案，不降级为摄像头扫码。

**Q3.6：产品计数功能？**
> **A：暂不需要。** 按产品区分扫码数量和扫码总数的计数功能暂不实现。

**Q3.7：扫码失败（条码无效/不存在）时如何处理？**
> **A：弹出选择框**，提供两个选项：
> - "继续扫描"：不关闭扫码，激光继续扫描下一个码
> - "停止扫描"：与点击"停止扫码"按钮逻辑一致，停止扫描并恢复所有按钮初始状态
> - 重复码也采用相同逻辑（选择框而非简单确认）

### 打印相关

**Q4：打印按钮位置？**
> **A：头部右侧**（和出库/删除按钮同一行）。但已出库/已退库状态下出库/删除按钮已隐藏（它们只在"制单"状态显示），所以打印按钮是那行唯一的按钮。

**Q5：打印触发条件？**
> **A：只在已出库、已退库状态才显示打印按钮。**
> - 已确认：当前代码中出库/删除按钮只在"制单"状态显示（`isEditable = status == "制单"`）
> - 已出库/已退库状态下出库/删除按钮不可见，打印按钮正好替代显示
> - 具体判断：`status == "已收货" || status == "已退库"`（注：后端返回"已收货"，前端显示为"已出库"）

**Q6：小票是否需要打印二维码？**
> **A：不需要。** 只打印文字明细，不打印二维码。

**Q7：打印表头包含哪些字段？**
> **A：单据编号、客户、日期、类型、备注**，全部打印。操作人/审核人等信息不打印。

**Q8：打印明细包含哪些字段？**
> **A：明细显示的内容全部打印**（序号、箱码、产品名称、规格、规格2），需要有列名（列名和系统里的名字一致）。

**Q9：纸宽能否自动识别 58mm/80mm？**
> **A：可以自适应。** V3PLUS 同时支持 58mm 和 80mm 纸卷（最大支持 80mm，兼容 58mm），代码使用比例排版（`colsWidthArr` 按比例分配宽度），58mm/80mm 都能自动适配，无需手动判断纸宽。58mm 可打印宽度约 384 像素，80mm 约 576 像素。

### 开发顺序

**Q10：先做哪个？**
> **A：先做扫码，测试通过后再做打印。**

**Q11：网络接口 getcode / addByCode / getALLCode 需要重新写吗？**
> **A：不需要。** 遵守"后端 0 改动"规则，当前 Android app 已有对应的网络请求代码，直接复用，不对照小程序重写。

**Q12：AIDL 文件从哪里来？**
> **A：以官方推荐为准。** 可从本地 SunmiHotel Demo 复制，也可手写创建，以官方文档推荐的方式为准，允许优化。

**Q13：测试方式？**
> **A：开发完成编译 APK 后，由用户在真实 V3PLUS 设备上测试。** AI 无法在真机上运行，仅负责代码编写和编译。
